# Repository Privacy

Public repository content and distributable artifacts must not expose personal or business identities, usernames, home-directory names, or machine-specific checkout paths. The permitted public project identities are `Eric Wimp` and `ericwimp8`.

After cloning, run `scripts/setup-repository-hooks.sh`. Every commit must pass the repository privacy hooks for staged content, tracked filenames, commit identities, and commit messages.

Use repository-relative project paths. In Markdown, never record machine-specific absolute paths; use `~` only when an external home-relative path is necessary.

Build every distributable Go CLI with `-trimpath` via `cli/scripts/build-cross-platform.sh`. Before publication, verify binaries contain neither the absolute checkout path nor non-permitted personal information.

Derive required absolute test paths from test-owned temporary directories or identity-neutral fixtures. Tests unrelated to paths must use non-path values; never hard-code machine-style test paths.
