# Service Release Guide

## Release Authorization

All releases, including hotfixes, require approval from two service owners who are not deploying the change.

## Deployment Steps

1. Confirm the build is reproducible.
2. Confirm approval from two service owners who are not deploying the change.
3. Deploy the signed artifact.

## Hotfixes

Hotfixes follow the same authorization requirements. Hotfix artifacts must still be signed.
