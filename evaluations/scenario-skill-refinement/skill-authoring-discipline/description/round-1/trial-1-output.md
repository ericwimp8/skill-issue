# Readiness Assessment

**Status:** Not ready to ship.

The frontmatter description is a broad list rather than the required concise what-it-is sentence followed by a when-to-use sentence. The body then repeats activation guidance and reduces release-note work to an exhaustive checklist of inputs, categories, and output sections. That structure encourages mechanical completion even when sources or sections are irrelevant, while omitting the governing behavior: evidence-backed claims, audience and release scope, user impact, and explicit handling of uncertainty and breaking changes.

## Smallest Coherent Changes

1. Replace the description with two concise sentences: identify the skill as release-note drafting guidance, then state that it applies when creating or reviewing release notes, changelogs, or version announcements.
2. Remove the body sentence that repeats when to use the skill.
3. Replace the itemized checklist with a few imperative rules that require establishing audience and release scope, deriving claims from available authoritative change evidence, prioritizing user impact, clearly surfacing breaking changes or migrations, and including only relevant sections.
4. Require uncertain or unsupported claims to be omitted or explicitly flagged rather than inferred.

These changes preserve the intended capability while making the skill concise, adaptable, and behavior-changing.
