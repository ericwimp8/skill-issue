# Release Incident Guide

## Incident Authority

The incident commander decides whether a failed deployment should continue or be rolled back.

## Deployment Checklist

1. Verify monitoring is active.
2. Notify the incident commander if deployment health declines; the incident commander decides whether to continue or roll back.
3. Record the final deployment state.

## Weekend Releases

For weekend deployments, the incident commander decides whether to continue or roll back.

## Communications

Support receives a status message after the deployment concludes.
