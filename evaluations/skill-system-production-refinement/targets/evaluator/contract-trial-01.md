# Contract Trial 01: Skill Evaluation and Refinement

## Goal

Reliably validate and, when evidence shows a material failure, semantically refine an existing skill through two ordered loops: first its proactive-invocation description, then its loaded body behavior.

## Intended Use

Use this workflow for an existing skill in a model-and-harness environment already qualified for reliable proactive skill invocation. Description trials additionally require an implicitly invocable packaged target, fresh independent agents or equivalent isolation, and native evidence that distinguishes skill loading from ordinary reasoning. Body evaluation applies to any contract-owned observable surface the harness can execute and inspect, or to externally executed prepared inputs when the current harness cannot.

## Expected Behavior

The agent must establish an evaluation-owned campaign, read the whole target and its contract-bearing references, interpret the target without changing its meaning, and keep description and body evaluation separate. It must pass the description loop first using two representative trials followed by two different confirmation trials, each run by a fresh agent and proven by native skill-load evidence. It must then derive observable body criteria, select automatic or review-before-update refinement mode, run varied isolated cases against minimal connected fixtures, audit every result, and translate retained failures into the smallest coherent change at the semantic owner. Each failed description round restarts the full four-trial protocol after cleanup; each failed body cycle is cleaned and rerun. At five unsuccessful rounds or cycles, it must stop at the specified scope and request explicit authorization for further attempts. It must conclude by recording either passing evidence or the exact blocking gate, retained evidence, cycle count, and user-controlled stopping state.

## Observable Expected Result

A successful campaign has native evidence for four proactive-invocation successes and behavior-audit evidence that every required body case and completion criterion passes with no retained material failure. Any resulting target refinement is generalized, reconciled with related meanings, and free of fixture-specific wording or duplicated semantic ownership; evaluation-owned transient outputs from prior cycles are removed while the campaign record and evidence remain. If the environment or a loop cannot qualify or pass, the observable result is a documented stop rather than an unsupported success claim.

## Preserved Boundaries

- Do not use the workflow to qualify an unproven model-and-harness environment; configuration capability is not measured invocation reliability.
- Do not run description trials for an explicit-only target, without isolated agents, or without native load evidence; agent self-report and output similarity are insufficient.
- Do not begin body evaluation before the description loop passes, merge the loops, or count partial trial success as a pass.
- Do not widen, narrow, add, or remove target meaning for evaluation convenience, literal expected answers, fixture names, or one failed example.
- Keep refinements at the meaning's normal semantic owner, preserve unrelated meaning, and apply the same constraints to automatic edits and review proposals.
- Preserve user control over refinement mode and over attempts beyond the five-round or five-cycle limits.
- Preserve evidence-level claims, campaign ownership, clean-fixture reruns, and external-execution waiting when the current harness cannot produce reliable evidence.

## Ambiguity

- The contract does not define the qualification procedure, threshold, or durable record that proves an environment is “already qualified.”
- “Representative,” “varied,” “material failure,” and sufficient body completion criteria remain target-dependent judgments rather than quantified standards.
- The exact campaign record schema, cleanup inventory, and boundary between retained evidence and transient outputs are unspecified.
- For externally executed body cases, the contract specifies reliable evidence and waiting, but not the exact evidence format for every possible surface.
