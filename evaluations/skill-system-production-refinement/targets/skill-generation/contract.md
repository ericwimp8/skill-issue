# Skill Generation Evaluation Contract

## Goal

Execute a complete Skill Intake plan into an idiomatic agent skill under its recorded viability, execution preference, authority, destination, and harness requirements, then produce a correct evaluation handoff.

## Expected Behavior

- Accept complete intake contracts without rediscovering intent; return only for intent-changing gaps.
- Map instruction, reference, script, asset, code, metadata, and packaging ownership before writing.
- Apply the required semantic and authoring disciplines.
- Create a concise canonical skill with only necessary resources and accurate host-specific metadata.
- Honor user stops and unavailable inputs without inventing support.
- Validate structure and each intake criterion while keeping runtime proof deferred.
- Hand the generated result to Evaluation with no shadow plan.

## Boundaries

Generation does not perform Intake again, create a second plan, over-package resources, claim unsupported harness capability, infer authority beyond the contract, or treat structural validation as behavior proof.
