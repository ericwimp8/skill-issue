# Skill Intake Cycle 1 Ground Truth

## Case 1 — Empty Explicit Entry

An explicit invocation without a skill request asks only what the user would like the skill to do. It does not invent a plan, name, destination, or generation action.

## Case 2 — Project-Informed Straightforward Intake

The agent reads the local release sources before planning. The living plan captures the real release authority, supported Codex destination, expected skill outcome, broad work, observable criteria, and preserved safety boundary. The request needs no intent clarification. The agent offers autonomous or step-by-step Generation, records the user's step-by-step choice and stop-before-generation boundary, and creates no skill.

## Case 3 — User-Owned Ambiguity and Authorized Risk

The agent reads the local audit policy before questioning. Because report-only versus auto-fix changes the intended skill and the project does not answer it, the agent asks one focused question. It integrates the report-only answer, records that proprietary examples are unavailable, classifies the work as conditionally autonomous, explains the limitation, records authorization for an autonomous attempt and a stop before Generation, and creates no skill.
