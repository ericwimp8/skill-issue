# Skill Intake Evaluation Contract

## Goal

Turn an explicit user request for a new or revised agent skill into one build-ready, project-informed A-to-B plan and a clear Generation handoff without creating the requested skill.

## Intended Use

The user explicitly invokes Skill Intake either with an initial skill description or before providing one.

## Expected Behavior

- Support both entry patterns and maintain one living plan.
- Organize free-form dictation into A, B, a broad dependency-ordered path, and observable C without duplicating meaning.
- Investigate source, local documentation, configuration, tools, and authoritative platform material before asking the user.
- Decide evidence-supported and ordinary implementation matters; ask only focused questions whose answers would materially change intent.
- Surface material omissions while leaving execution choices open.
- classify autonomous, conditionally autonomous, or ongoing-participation mode; explain the assessment; record the user's choice, stops, and any risk authorization.
- Produce the durable plan and compact Generation handoff.

## Expected Result

Another agent can generate the intended skill without rediscovering user intent or known project context. The handoff names destination, mode, authority, stops, external dependencies, outcomes, completion criteria, and supported harnesses when known.

## Boundaries

- Intake remains explicit-only.
- Intake does not create or refine the requested skill.
- Project-answerable questions stay with the model.
- Ordinary generation choices do not become user questions.
- The A-to-B plan remains the source of truth; no shadow specification is created.
