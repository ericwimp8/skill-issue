# GitHub Copilot sub-agent launch behavior

1. **Does GitHub Copilot support sub-agents?** Yes. GitHub documents subagents as separate agents spawned by the main Copilot agent to handle delegated work in isolated contexts. This is supported in current Copilot surfaces including VS Code and Copilot CLI. [GitHub Copilot customization cheat sheet](https://docs.github.com/en/copilot/reference/customization-cheat-sheet#usage-comparison)

2. **If yes, how is a sub-agent launched?** The main agent launches it through model-driven delegation. Copilot may decide automatically, or the user can request a subagent in the prompt; in VS Code the `agent/runSubagent` tool must be enabled. Copilot CLI also provides `/fleet` to have the main agent split a prompt and run suitable subtasks through subagents. [VS Code subagent invocation](https://code.visualstudio.com/docs/agents/subagents#_invoke-a-subagent) · [Copilot CLI `/fleet`](https://docs.github.com/en/copilot/concepts/agents/copilot-cli/fleet)

3. **Can a skill launch or request a sub-agent?** Yes, with surface-specific support. In VS Code, a skill can set experimental `context: fork` in `SKILL.md`; Copilot then executes that skill in a dedicated subagent and returns only its final result to the parent. Because ordinary skill instructions are loaded into the parent agent's context, they can also request delegation, but the main agent and its enabled subagent tool remain the launch mechanism. [VS Code agent skills: forked context](https://code.visualstudio.com/docs/agent-customization/agent-skills#_run-a-skill-in-a-forked-context) · [VS Code subagent launch mechanism](https://code.visualstudio.com/docs/agents/subagents#_how-subagents-are-invoked)
