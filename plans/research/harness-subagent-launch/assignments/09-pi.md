# Pi Sub-Agent Launch

1. **Does Pi support sub-agents?** Yes, through Pi's official `subagent` extension example, which adds sub-agent delegation as a tool. [Pi subagent extension source](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/examples/extensions/subagent/index.ts)

2. **If yes, how is a sub-agent launched?** The active agent calls the extension-registered `subagent` tool; a single launch supplies `{ agent: "name", task: "..." }`. The extension starts a separate `pi` process in JSON/print mode with no session, giving the sub-agent an isolated context. [Pi subagent tool registration and parameters](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/examples/extensions/subagent/index.ts#L448-L470) [Pi child-process launch](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/examples/extensions/subagent/index.ts#L294-L339)

3. **Can a skill launch or request a sub-agent?** A skill can request delegation by instructing the active agent to call the available `subagent` tool. Pi loads a skill as instructions that the agent follows; the extension tool performs the actual process launch. [Pi skills: how skills work](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/docs/skills.md#L64-L69) [Pi subagent tool source](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/examples/extensions/subagent/index.ts#L460-L470)
