# Task 1: Establish the Target Skills' Intended Outcomes

## A: Starting Position

- Three Eric Wimp Toolkit skills have been copied locally as test targets: `document-update-discipline`, `prompt-writing`, and `dictate-plan`.
- Each complete target skill is available for inspection under `test-targets/skills/`.
- The target set provides short, medium, and longer workflow-oriented skill bodies for later evaluation and refinement.

## B: Desired Outcome

Each target skill has an evaluation-ready interpretation that clearly establishes its goal, intended use, expected behavior, and expected result without extending or narrowing its actual contract.

## Path from A to B

### 1. Read Each Complete Target Skill

Treat each target's frontmatter and complete body as one coherent source of meaning.

### 2. Establish Each Skill's Goal

Record the outcome each skill exists to produce, using an explicitly stated goal when available and deriving it from the complete skill when it is unstated.

### 3. Establish Each Skill's Intended Use

Record the situations in which each skill is intended to apply without expanding or narrowing its stated invocation boundary.

### 4. Establish Each Skill's Expected Behavior

Distinguish the behavior each skill requires after invocation from the result that behavior is meant to produce.

### 5. Establish Each Skill's Expected Result

Define an observable successful end state for each target that can ground its later description and skill-behavior evaluations.

## C: Completion Criteria

- All three target skills are identified unambiguously by local path and canonical source.
- Each target's goal, intended use, expected behavior, and expected result are established from its complete contents.
- Each interpretation preserves the target's actual contract without adding evaluation-driven meaning.
- The three interpretations are detailed enough to ground the later description and skill-behavior evaluations.
