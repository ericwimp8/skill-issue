# Local GUI Finishing Editors for macOS

## Assignment

**Goal:** Select the best no-cost, local GUI path for finishing a polished public 1080p YouTube demonstration under three minutes on macOS, comparing iMovie, Shotcut, OpenShot, DaVinci Resolve Free, Kdenlive, and native Screenshot/QuickTime capture.

**Scope:** Current macOS availability and requirements; clean 1080p export; watermarks; titles, intro/outro, transitions, audio, captions, capture, speed/minimalism, practical learning time, automation fit with Codex/computer-use and generated assets/instructions, output quality, licensing, privacy, download/account friction, and estimated hands-on time. Research date: 2026-07-19.

**Exclusions:** Paid GUI products, browser editors, code-first renderers, purchase/installation trials, and a claim that an editor can be reliably driven end-to-end by desktop automation.

## Findings

### Decision: Screenshot + iMovie Is the Best Simple Local GUI Path

For this exact short public demo, use macOS Screenshot for capture and iMovie for the final cut. Screenshot is already present on macOS Mojave or later and records an entire screen or selected portion, microphone audio, visible click rings, a timer, and a locally saved movie; QuickTime is an equivalent capture entry point on older UI flows. iMovie accepts that local capture and provides the needed title cards, transitions, music/sound clips, voiceover recording, fades, audio enhancement, and video export. Apple documents 1080p60 output on Mac and says 1080p is appropriate for most social sites; iMovie can save the finished movie as a local file for later YouTube upload. [Apple Screenshot capture](https://support.apple.com/en-us/102618) and the [iMovie guide](https://support.apple.com/en-euro/guide/imovie/welcome/mac) establish the capture and finishing feature chain; [Apple export guidance](https://support.apple.com/en-us/102371) and [1080p60 support](https://support.apple.com/en-gb/102276) establish the delivery target.

**Evidence:** Apple’s capture path runs locally and its guide does not require a recording-service account. iMovie’s documented macOS workflow is a simple timeline with built-in titles, transitions, music, voiceover, audio correction/effects, and 1080p60/4K support. Screenshot capture can show pointer clicks and saves to a chosen local folder; QuickTime can also export a local 1920×1080 H.264/HEVC MOV. [Apple Screenshot](https://support.apple.com/en-us/102618), [iMovie guide](https://support.apple.com/en-euro/guide/imovie/welcome/mac), and [QuickTime 1080p export](https://support.apple.com/en-ie/guide/quicktime-player/qtp20e395859/mac).

**Implication:** This is the no-cost baseline with the least setup and privacy surface: record a 16:9 selected screen region, trim mistakes, place a 3–5 second iMovie title card before it, add a 3–5 second closing card, use only one restrained transition type, balance narration/music, and export a local 1080p file. The estimated user hands-on time is 30–60 minutes after a prepared script/assets, or 60–90 minutes if re-recording is needed. That is an estimate from the deliberately small editing recipe, not a vendor performance claim.

### Recommended Minimal Production Recipe

1. Have Codex produce a 90–150 second narration, shot list, exact title/outro copy, a timed SRT, and SVG/PNG logo/card assets. Keep the live product demo to one continuous flow with retakes as separate clips.
2. Use `Shift-Command-5` Screenshot to record only the 16:9 demo region, with microphone and click indicators enabled if useful. Record to a local project folder. [Apple’s controls and local-save behaviour](https://support.apple.com/en-us/102618).
3. In iMovie, place intro card → trimmed screen recording → optional one cross-dissolve → closing card; record/import narration, set fades and audio levels, then export the 1080p local file. [iMovie’s titles/transitions/audio workflow](https://support.apple.com/en-euro/guide/imovie/welcome/mac) and [local file export](https://support.apple.com/en-us/102371).
4. Watch the complete exported file locally before upload. Upload captions as the prepared SRT in YouTube Studio or create YouTube captions after upload; this avoids treating iMovie’s undocumented caption capability as established.

**Evidence:** Native capture supports a timer, selected-area capture, mic, click display, and an explicitly chosen local save location. iMovie documents the editing primitives and file export used in the recipe. [Apple Screenshot](https://support.apple.com/en-us/102618) and [iMovie export](https://support.apple.com/en-us/102371).

**Implication:** Codex supplies deterministic, reviewable pre-production assets and click-by-click instructions while the user retains responsibility for macOS permissions, capture performance, final visual/audio review, and public upload. This division achieves substantial Codex involvement without claiming unsupported control of a GUI timeline.

### Lower-Fit Selection Rules

Choose **Shotcut** instead of iMovie only when an open-source licence, no telemetry/activation, built-in screen-recording entry point, or a more flexible timeline is a concrete requirement. Its own FAQ confirms it needs no network for activation/subscriptions/analytics and its current macOS build supports 12–26. [Shotcut FAQ](https://www.shotcut.org/FAQ/) and [download page](https://www.shotcut.org/download/).

Choose **Kdenlive** only when local subtitle authoring/Whisper/VOSK processing, subtitle import/export, or generated render scripts materially outweigh its stricter macOS 13 Apple Silicon requirement and complex UI. [Kdenlive captions](https://docs.kdenlive.org/en/effects_and_filters/subtitles.html), [speech-to-text](https://docs.kdenlive.org/en/effects_and_filters/speech_to_text.html), and [macOS requirements](https://docs.kdenlive.org/en/getting_started/installation.html).

Choose **OpenShot** if its title templates, speed control, and 3D title effects add real value and the machine is an Intel macOS 10.15+ system. Its own requirements say downloadable binaries are Intel-only; that makes it an inferior default on an Apple Silicon Mac. [OpenShot features](https://www.openshot.org/features/) and [requirements](https://support.openshot.org/support/solutions/articles/43000625545-system-requirements).

Choose **Resolve Free** only for an experienced editor who has already accepted the registration/privacy friction and needs its color, Fairlight audio, or Fusion finishing depth. Its Free ceiling is ample for 1080p, but its breadth makes it an avoidable learning risk for a short demo. [Blackmagic download/registration](https://www.blackmagicdesign.com/in/event/davinciresolvedownload) and [Free feature limits](https://www.blackmagicdesign.com/products/davinciresolve/studio).

**Evidence:** These selection gates pair each tool’s documented distinctive capability with a documented requirement or workflow cost, rather than treating the existence of more features as an automatic win.

**Implication:** The final recommendation remains stable: start local with Screenshot + iMovie, retain Shotcut as the no-account/open-source fallback, and escalate to Kdenlive or Resolve only for a specific need that iMovie cannot meet.

## Notes

- **Unsupported/verify before reliance:** Exact current installer/download sizes were not published in the primary pages inspected for iMovie, Shotcut, Kdenlive, or Resolve; only OpenShot publishes a 1 GB minimum storage requirement. Treat storage as a device check, particularly for Resolve and media caches.
- **Unsupported/verify before reliance:** A primary source did not establish iMovie caption authoring/automatic captions, Shotcut captions, or the current Resolve Free caption boundary. The recommended route therefore supplies captions outside iMovie rather than making an unsupported claim.
- **Watermark caveat:** Open-source projects and native Apple tools do not document an export watermark gate, but the absence of a published restriction is weaker evidence than a real export. Make a ten-second 1920×1080 export using the exact selected filters/titles before the full cut; Resolve Studio-only effects are explicitly watermarked in the Free version.
- **Privacy caveat:** “Local” describes the editing/capture path, not every auxiliary asset, font, stock-media, AI transcription, or YouTube upload. Avoid recording sensitive material while signed into unrelated accounts, and review the selected tool’s current terms before enabling connected services.
