# Pi Manual Skill Invocation

## Assignment

**Goal:** Determine whether a user can manually invoke an installed Pi agent skill named `dictate-plan`, identify the exact supported action or syntax, and supply the strongest first-party explicit wording only if dedicated syntax is unavailable.

**Scope:** The current first-party Pi coding-agent product and its first-party repository documentation for manually invoking an already installed agent skill.

**Exclusions:** Skill installation, CLI rollout, sub-agents, evaluation evidence, plugins, packaging, permissions, and unrelated Pi behavior.

## Findings

### Finding 1: A user can manually invoke a specific installed skill

Yes. The relevant current first-party product is **Pi Agent Harness**, specifically its interactive coding agent CLI, `@earendil-works/pi-coding-agent`. Pi registers discovered skills as `/skill:name` commands and documents the skill-command setting as enabled by default.

**Evidence:** The first-party [Pi repository README](https://github.com/earendil-works/pi#pi-agent-harness) identifies the project as Pi Agent Harness and the coding-agent package as its interactive coding agent CLI. The [Skills documentation](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/docs/skills.md#skill-commands) states that skills register as `/skill:name` commands. The [Settings documentation](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/docs/settings.md#resources) lists `enableSkillCommands` with a default of `true` and describes it as registering skills as `/skill:name` commands.

**Implication:** For an installed skill whose declared name is `dictate-plan`, Pi provides a dedicated manual invocation mechanism by default. This is direct command expansion rather than reliance on the model to recognize a natural-language request.

### Finding 2: The exact dedicated action is `/skill:dictate-plan`

At Pi's interactive prompt, type and submit:

```text
/skill:dictate-plan
```

To include the initial dictation or task in the same submission, use:

```text
/skill:dictate-plan <your dictation or task>
```

**Evidence:** Pi's [Skills documentation](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/docs/skills.md#skill-commands) gives the forms `/skill:brave-search` for loading and executing a named skill and `/skill:pdf-tools extract` for loading a skill with arguments. It specifies that text after the command is appended to the skill content as `User: <args>`. Pi's [Extensions documentation](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/docs/extensions.md#input) independently records the normal input-processing order: unhandled `/skill:name` input is expanded as a skill command before agent processing begins.

**Implication:** Substituting the caller-supplied installed skill name into Pi's documented syntax yields `/skill:dictate-plan`. Trailing task text is supported, so the user may start the skill with concrete plan dictation in one action.

### Finding 3: No natural-language substitute is required

A dedicated syntax exists, so the requested no-syntax fallback is not applicable. Pi separately documents ordinary prompting as a way to force skill use, but it does not prescribe a canonical natural-language sentence. If the dedicated command has been disabled in local settings, the best-supported explicit wording is:

```text
Use the dictate-plan skill for this request: <your dictation or task>
```

That sentence is a conservative wording inferred from Pi's instruction to use prompting; it is not a reserved command or a first-party guaranteed equivalent to `/skill:dictate-plan`.

**Evidence:** In [How Skills Work](https://github.com/earendil-works/pi/blob/main/packages/coding-agent/docs/skills.md#how-skills-work), Pi says models do not always load a matching skill and tells users to use "prompting or `/skill:name` to force it." The same page defines the slash-command form precisely but supplies no exact prose template.

**Implication:** Use `/skill:dictate-plan` for explicit, dedicated manual invocation. Treat natural-language wording as model-mediated selection whose exact phrasing and outcome are not guaranteed by the reviewed first-party sources.

## Notes

- The exact command assumes the installed skill's declared name is `dictate-plan`, as supplied by the assignment. The first-party sources establish Pi's name-based invocation contract but do not independently verify a particular local installation.
- A user can turn off skill commands through `enableSkillCommands`; the first-party settings reference documents `true` as the default. When disabled, Pi documents prompting as an alternative but gives no canonical sentence.
