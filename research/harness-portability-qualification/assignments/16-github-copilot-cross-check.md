# GitHub Copilot CLI Cross-Check

## Assignment

**Goal:** Cross-check the current GitHub Copilot CLI evidence against primary GitHub and OpenAI sources, reconcile the first technical gate with the Codex-subscription second gate, and state the campaign decision.

**Scope:** The supplied local assignments; current Skill Issue production source; and current official GitHub and OpenAI documentation inspected 2026-07-21. The assessment covers only Copilot CLI as a potential Skill Issue evaluation harness.

**Exclusions:** Installing, authenticating, or executing Copilot; credential inspection or transfer; proxy creation; configuration mutation outside an output-owned run root; implementation design; and treating documentation as live qualification.

### Reconciled Gate Decision

The first technical gate is **blocked pending named evidence or access**: a pinned installed Copilot CLI, approved account or provider route, a private-home/cache launch, exact two-turn session continuity, strict JSONL success/error/session predicates, complete discovery inventory and exclusions, a noninteractive least-privilege permission result, descendant-reaping observations, and local-plus-remote retention evidence are absent.

The second gate is **blocked pending named evidence or access** for the requested Codex-subscription-only route: official BYOK is a separate provider route that needs a user-owned key or local model, and OpenAI API billing is separate from ChatGPT. Because the caller's campaign rule removes a candidate when this second gate fails, the evidence-backed decision is **remove from the supported campaign**.

### Unsupported Evidence and Rejected Alternatives

- Unsupported: an installed executable/version; native authentication or BYOK access; policy outcome; exact command compatibility; model and effort resolution; JSONL schema; machine-readable skill activation; deterministic resume capture; complete discovery exclusion; prompt-free least-privilege execution; process-tree cleanup; and remote-retention cleanup.
- Rejected: passing Codex OAuth, ChatGPT cookies, Codex CLI secrets, copied keychain material, or a copied Copilot home as a BYOK credential. GitHub documents provider API keys or local models, while OpenAI documents separate API billing.
- Rejected: installing or managing a loopback compatibility proxy, organization/enterprise custom-model configuration, permanent skill/plugin installation, or broad `--allow-all`/`--yolo` permission. Each either changes non-owned state, escapes the requested subscription route, or violates the evaluation safety boundary.

## Notes

- Current GitHub documentation has moved configuration toward `COPILOT_HOME`, `COPILOT_CACHE_HOME`, `settings.json`, and `--plugin-dir`; the current command reference did not substantiate `COPILOT_SKILLS_DIRS`. Treat that older variable claim as unsupported until a pinned executable's own help or primary documentation confirms it.
- `--no-remote-export` narrows documented export behavior, while the session-data page states default GitHub-account synchronization. This supports retaining the cloud-state uncertainty rather than claiming deletion of a local run root erases remotely synchronized data.
- The local executable absence is an access blocker only; the campaign removal follows the Codex-subscription second-gate failure, not the absence itself.
