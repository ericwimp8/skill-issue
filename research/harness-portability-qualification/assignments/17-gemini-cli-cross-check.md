# Gemini CLI Cross-Check

## Assignment

**Goal:** Cross-check the Gemini CLI portability evidence against current official Google Gemini CLI and OpenAI primary sources, resolve product-identity and feature/version conflicts, and decide both qualification gates.

**Scope:** Assignments 01, 02, 03, 05, 07, and 12; current official Gemini CLI documentation, Google’s 19 May 2026 transition announcement, the official `google-gemini/gemini-cli` release record, and official OpenAI documentation.

**Exclusions:** Installation, authentication, credential inspection or transfer, direct Gemini execution, source changes, provider/proxy setup, and an implementation plan.

## Findings

### Product Identity Supersedes The Earlier Consumer-Candidate Assumption

Gemini CLI remains a published CLI with a current stable `v0.51.0` release, but its supported-access population changed materially on 18 June 2026. Google states that Gemini CLI stopped serving Google AI Pro/Ultra and no-charge Gemini Code Assist for Individuals then. It retains Gemini CLI service for Gemini Code Assist Standard/Enterprise and Google Cloud users, and says paid Gemini and Gemini Enterprise Agent Platform API keys retain access. Antigravity CLI is a separate terminal product; Google describes feature migration but does not establish Gemini CLI invocation, sessions, state, or evidence as Antigravity CLI contracts.

**Evidence:** Google’s announcement names the consumer stop date and the retained enterprise/API-key routes. The official release record and the current docs still describe `gemini`, skills, JSONL headless output, and session management. Assignment 03 correctly marks local `gemini` unavailable, but Assignment 07’s consumer-oriented route and Assignment 12’s broad Google-login framing require this eligibility correction.

**Implication:** For the caller’s existing paid OpenAI Codex/ChatGPT-only access requirement, the candidate is **remove from the supported campaign**. A future, separately scoped Gemini evaluation may be considered only for an operator-owned eligible Google enterprise/API-key route; it starts as **blocked pending named evidence or access**, not as a continuation of this campaign. Antigravity CLI is a conditional separate candidate and cannot inherit this Gemini record.

### Conditional And Rejected Alternatives

An operator-owned Gemini Code Assist Standard/Enterprise, Google Cloud, paid Gemini, or Gemini Enterprise Agent Platform API-key route is conditionally relevant only to a newly scoped, Google-entitled campaign. It still requires the first-gate evidence matrix above and must preserve the operator’s credentials and configuration. The official Google announcement is the controlling product-identity source for this condition.

**Evidence:** Google retains Gemini CLI service for named enterprise/API-key routes while directing consumer users to Antigravity CLI. Gemini’s plans and authentication pages distinguish Google account, AI Studio API key, and Vertex/Cloud access. The local contract marks Antigravity CLI as a separate surface.

**Implication:** The conditional alternative is **blocked pending named evidence or access**; it does not reopen the present campaign. Antigravity CLI must receive its own product, invocation, state, protocol, and access audit.

Third-party Gemini-protocol translation gateways, OpenAI API-key routes, copied Codex/ChatGPT OAuth or credential caches, persistent edits to user/project settings or trust files, and broad `--approval-mode=yolo` are rejected alternatives.

**Evidence:** Gemini’s supported authentication does not include OpenAI credentials; OpenAI’s billing is separate; Gemini’s gateway/base-URL configuration is not an OpenAI-provider contract; policy docs show `yolo` broadly allows tools. Assignment 12 already traces the lack of a native OpenAI content-provider route.

**Implication:** These alternatives cannot change the decision from **remove from the supported campaign** and would create unsupported entitlement, privacy, or permission claims.

## Notes

- The older `v0.51.0` claim is currently corroborated by the official GitHub Releases API, but every automation option remains version-sensitive and must be proved against the selected executable; source/docs alone do not qualify it.
- Google’s current documentation still contains consumer-plan and Google-account wording while displaying a transition banner. The dated Google announcement resolves the conflict: after 18 June 2026, those consumer routes no longer receive Gemini CLI requests.
- The matrix deliberately distinguishes a technical route that could be examined with independent eligible Google access from the caller’s Codex-subscription requirement. The latter has a final second-gate failure.
