# Google Antigravity and Gemini CLI Sub-Agent Launch

1. **Does the selected Google harness surface support sub-agents?** Yes. Both current Google Antigravity and Gemini CLI support sub-agents. Antigravity runs asynchronous sub-agents that report to a parent agent; Gemini CLI runs specialized agents in isolated context loops within the main CLI session. [Google Antigravity: Asynchronous Subagents](https://antigravity.google/docs/subagents) [Gemini CLI: Subagents](https://geminicli.com/docs/core/subagents/)

2. **If yes, how is a sub-agent launched?** In Antigravity, the parent agent calls the `invoke_subagent` tool with one or more role-and-prompt specifications; the agent can define a custom role first with `define_subagent`. In Gemini CLI, each available sub-agent is exposed to the main agent as a same-named tool, which the main agent calls automatically when appropriate; a user can strongly request a particular one by starting the prompt with `@<subagent-name>`. [Google Antigravity: Invoking Subagents](https://antigravity.google/docs/subagents#invoking-subagents) [Gemini CLI: Using subagents](https://geminicli.com/docs/core/subagents/#how-to-use-subagents)

3. **Can a skill launch or request a sub-agent?** Yes, as an instruction to the main agent rather than as an independently executing launcher. On both surfaces, a skill supplies instructions that the agent follows, so it can direct the agent to use the surface's sub-agent tool; the actual launch remains the parent agent's `invoke_subagent` call in Antigravity or named sub-agent tool call in Gemini CLI. Neither current skill format documents a `context: fork`-style field that launches a sub-agent by itself. [Google Antigravity: Agent Skills](https://antigravity.google/docs/skills) [Google Antigravity: Subagent tool](https://antigravity.google/docs/subagents#invoking-subagents) [Gemini CLI: Activate skill](https://geminicli.com/docs/tools/activate-skill/) [Gemini CLI: Subagent tool model](https://geminicli.com/docs/core/subagents/#what-are-subagents)
