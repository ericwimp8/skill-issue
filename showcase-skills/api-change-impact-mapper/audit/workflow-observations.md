# Workflow Observations

## Fresh-Agent Capacity

The governed campaign required seven fresh independent agents. Shared team capacity repeatedly prevented new trials from starting even though their inputs were ready. The campaign preserved the fresh-agent requirement by waiting and sequencing work rather than substituting inherited or same-agent evidence. A campaign scheduler that reserves or queues required trial slots would reduce idle time without weakening isolation.

## Environment Evidence Shape

The repository retains medium-reasoning Codex qualification across the CLI setup, smoke report, and completed showcase evidence rather than in one campaign-local qualification document. Each trial retained its fresh agent identity, exact target hash, and pre-output read requirement, but the collaboration surface exposed no separate native session identifier. The evaluation workflow would be clearer if it named an accepted campaign-local evidence schema for fresh collaboration agents and distinguished it explicitly from CLI marker attribution.

## Formatting and Evidence Immutability

The repository's required formatting command does not traverse showcase evaluation outputs or fixtures. A broader optional Prettier check can therefore identify changes after trial hashes are frozen. Formatting ownership should be explicit: either normalize all campaign-owned inputs before trial hashing and retain raw agent outputs as evidence, or define a derived formatted presentation copy whose hash does not replace the raw output.

## Reference Protocol

The generated skill required no references, so the per-reference qualification and post-refinement routing protocol were correctly not applicable. Avoiding an unnecessary reference removed eight otherwise required fresh traversal trials and kept one semantic owner for the analysis behavior.
