# Workflow Observations

## Fresh-Agent Capacity Gate

The governed evaluation requires many fresh independent agents, while the shared team has a fixed concurrency limit and other showcase campaigns run simultaneously. The workflow says to wait when agents are unavailable but provides no reservation, queue, or campaign scheduling mechanism. This can leave a fully generated target and qualified campaign idle between trials. A future harness-level queue could preserve freshness and ordering without encouraging evidence substitution.

## Candidate-Selection Evidence Boundary

The available in-process surface can retain exact candidate-description and selected-skill pre-output reads, but it does not prove opaque native automatic injection. The campaign therefore qualifies and claims only candidate-selection reliability. The evaluation workflow would benefit from explicitly naming this lower evidence class and stating whether it is sufficient for showcase description evaluation when native runner execution is unavailable or approval-bound.

## External Source Drift

Dependency migration guidance can change or move after an evaluation. Trial records should retain source URLs, access dates, version applicability, and the project facts used to determine relevance. The workflow requires authoritative sources but does not explicitly define a minimum durable evidence record for drift-prone upstream pages.
