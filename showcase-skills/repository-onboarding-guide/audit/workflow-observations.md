# Workflow Observations

- The workflow kept intake intent, generation structure, and evaluation evidence under one bounded showcase workspace without changing production workflow owners.
- Global fresh-agent capacity allowed only two body cases to start simultaneously; the third started immediately when a slot released. Every case still used a fresh GPT-5.6 Sol medium agent with `fork_turns: "none"` and remained isolated from sibling output.
- Independent agents occasionally executed additional bounded diagnostics. Requiring them to label project-owned, ad hoc, successful, failed, and environment-inconclusive commands kept those probes from becoming unsupported contributor conventions.
- The target needed no packaged references, scripts, or assets. Avoiding them eliminated reference-routing qualification work and kept the portable skill surface concise.
- The sub-agent interface does not expose a standalone exported native tool transcript to the parent. Each independent trial therefore retained exact pre-output candidate or target reads, hashes, commands, and results inside its owned evidence log; the project-local CLI remains the stronger route when structured skill-activation events are required as the sole evidence form.
- The repository-owned `npm run format:check` gate passed but its configured glob does not include showcase workspaces. A direct expanded Prettier probe flags existing showcase-style wrapping, so future workflow owners may want an explicit showcase formatting contract rather than relying on the website-only command.
