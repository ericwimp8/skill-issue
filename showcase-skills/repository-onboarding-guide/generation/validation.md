# Generation Validation

- Skill folder and `name` use matching lowercase hyphenated form: pass.
- Frontmatter contains only `name` and a concise purpose/use-boundary description: pass.
- Body contains behavior-changing instructions and no activation section: pass.
- Codex metadata contains only requested interface fields and no default prompt: pass.
- References, scripts, and assets are absent because the workflow is concise, contextual, and non-deterministic: pass.
- Every intake completion criterion is represented in the generated contract: pass.
- Runtime-dependent criteria are handed to evaluation rather than claimed from structure: pass.
