# Generation Handoff Contract

## Required Intake Inputs

- Build-ready A-to-B plan.
- Confirmed project and platform context.
- Intended skill purpose, outcome, and invocation boundary.
- Observable completion criteria.
- Destination and supported harness surfaces.
- Unresolved implementation-time matters.
- User-owned stop conditions.
- Generation viability.
- Selected execution preference.
- Authority boundary.

If a missing item changes intent, return to intake. If it affects only implementation, investigate and decide within the recorded authority.

## Required Evaluation Outputs

- Generated skill path.
- Supported and unsupported harness surfaces.
- Goal, intended use, expected behavior, expected result, and preserved boundaries.
- Runtime completion criteria.
- Known limitations and unresolved capabilities.
- Refinement mode when already chosen.
- Next action: continue directly into evaluation or stop at the exact missing prerequisite or user-owned boundary.

Keep the original intake plan as the source of truth. Do not create a second shadow plan during generation.
