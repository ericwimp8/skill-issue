# Document Update Discipline

## Goal

Prevent **proximal patching** in a document: changing the nearest visible
mention instead of integrating meaning at the semantic location that owns it.

A local edit is correct when that place owns the idea. The failure is choosing
the edit site from proximity or visibility before establishing ownership.

False inference: the request pointed here, therefore the change belongs here.  
Accurate inference: the request pointed here, so find where the meaning is owned.

**Core principle:** Observation starts investigation. Semantic ownership decides
where the document is updated.

Once loaded, this workflow is mandatory. Complete it before editing.

## Required Decision

Before editing, state:

- the meaning that must hold after the update,
- the document's purpose and structure,
- the observation point vs the semantic home,
- every related place that duplicates, implies, or contradicts that meaning,
- whether to add, remove, replace, move, consolidate, or rewrite,
- which unrelated meanings must stay intact.

If you cannot state these, keep reading before editing.

## Workflow

1. **Preserve the request without adopting its edit site.** Separate the
   meaning needed, where the user noticed the gap, and any suggested wording or
   insertion point.

2. **Understand the whole document** unless the user limits scope and accepts
   the risk. Know purpose, structure, existing meaning for the topic, and
   related authoritative wording before changing anything.

3. **Separate locations:** observation point, semantic home, related
   manifestations, and unrelated meanings that must not change. Do not assume
   the observation point is the home.

4. **Establish the semantic owner.** Ask which location owns this meaning for
   the reader, whether the apparent site is only a mention or convenient
   heading, and whether adding would create a second owner instead of
   replacing, moving, or consolidating. A local edit is correct only when that
   place owns the meaning.

5. **Make the smallest complete update** at the semantic home, not by
   proximity. Integrate there; keep related ideas together; update or remove
   every related manifestation; and resolve conflicts. Preserve every material
   meaning, distinction, qualifier, uncertainty, and relationship. Remove
   meaning only when it is explicitly superseded or intentionally deleted.
   Consolidation, restructuring, and rewriting must not cause semantic loss.
   Match existing tone and structure; state current truth directly (no
   "previously X, now Y" unless the document is historical). Reject patches
   that only insert at the observation point or leave duplicates/conflicts
   behind.

6. **Reconcile, then verify.** After editing the home, check related sections,
   examples, and cross-references. Confirm the meaning lives in the right
   place(s), conflicts are gone, unrelated ideas are intact, and the document
   reads as intentionally updated—not merely patched where it was pointed.

## Stop And Reframe

Return to mapping when the edit is justified by convenience, only the visible
mention changed, ownership is still unclear, unrelated ideas were weakened, or
user pushback shows the framing is wrong.
